# v0.2.0 Release Notes

## Highlights

- Telegram-origin turns now show live `Current tool:` from App Server `item/*` events while the tool is running.
- After completion, `[Tool]` returns to `Last completed tool:` and `[Output]` shows the completed output.
- Foreign GUI/CLI observer panels keep the safer completed-tool-only behavior from v0.1.3.
- App Server session handling is generation-guarded so stale events from a closed process cannot update the new session.

## Notes

- No session-tail/JSONL live overlay was reintroduced.
- No App Server protocol changes were required.
- Broker process and reattach-after-daemon-death remain future work.
